## 🎯 About This Portfolio

This repository showcases how I approach **messy real-world data**:

- ✅ **Translate ambiguous requirements** into measurable, actionable logic
- ✅ **Design fair, scalable frameworks** that account for real-world complexity
- ✅ **Build analytics solutions** that drive manager and leadership decisions
- ✅ **Document analytical thinking** (not just outputs) — showing the reasoning behind decisions

### What You'll See Here

Each project includes:
- Clear problem statement and business context
- Analytical approach and key challenges addressed
- Technical implementation details
- Visual outputs and key findings
- Design decisions and trade-off analysis

*Note: All materials have been redacted/simulated to remove proprietary information while preserving analytical integrity.*

## 🚀 Getting Started

### Prerequisites
- Python 3.8+
- Jupyter Notebook
- Power BI Desktop (for dashboard project)

### Setup Instructions

See **[SETUP.md](docs/SETUP.md)** for complete environment setup and how to run the analysis notebooks.

Quick start:
```bash
git clone https://github.com/icivoryable/ian-ivory-operations-analytics-portfolio.git
cd ian-ivory-operations-analytics-portfolio
pip install -r requirements.txt
jupyter notebook
```

## 📊 Key Analytical Themes

### 1. **Performance Measurement Under Uncertainty**
How do you fairly evaluate performance when timelines, start dates, and data collection methods vary? This portfolio demonstrates structured approaches to:
- Defining multiple valid KPI interpretations
- Setting equitable performance benchmarks
- Communicating trade-offs to stakeholders

### 2. **Translating Business Ambiguity into Data Logic**
Real business problems are rarely clean. Projects showcase:
- Converting stakeholder needs into measurable requirements
- Handling conflicting business rules
- Designing scalable data models

### 3. **Storytelling with Data**
Beyond dashboards and reports:
- Visual design for different audiences (managers vs. leadership)
- Prioritizing insights for impact
- Documenting assumptions and limitations
